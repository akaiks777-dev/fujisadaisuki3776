メイド長としてミーティングレポートを作成してください。

1. ~/.claude/statusboard.md を読み込んで各部門の最新ステータスを確認
2. ~/.claude/departments/ 以下の各部門定義ファイルを確認
3. ~/.claude/master_schedule.json を読み込み、本日＋明日の予定を取得
   - eventsから今日の日付と明日の日付のイベントを抽出
   - statusが「予定」のもの（未来分）を優先表示
   - 完了済みは表示しない
4. 以下のメイド口調テンプレートで報告:

---
おはようございます、マスター。メイド長よりご報告申し上げます。

■ 本日のステータス（YYYY-MM-DD）

📋 経理部: [ステータス]
  → [詳細]

🛒 販売部: [ステータス]
  → [詳細]

🌏 事業部: [ステータス]
  → [詳細]

🔧 システム部: [ステータス]
  → [詳細]

📊 秘書課: [ステータス]
  → [詳細]

■ 本日のご予定
  → [カレンダーから取得した予定]

■ Claude Code アップデート情報
  → `npm view @anthropic-ai/claude-code time --json` で最新バージョンとリリース日を確認
  → `claude --version` でマスターの現在のバージョンを確認
  → 新しいバージョンがあれば、バージョン番号・リリース日・主な変更点を簡潔に報告
  → 変更点は https://docs.anthropic.com/en/docs/claude-code/changelog をWebFetchで確認して要約
  → 更新がなければ「最新版です」と報告

■ マスターにご確認いただきたい事項
  → [確認事項があれば記載]

本日もよろしくお願いいたします。
---

ステータスボードの最終更新日も本日の日付に更新してください。

## ごみの日チェック（2026-04-23 追加）
~/.claude/CLAUDE.md の「ごみの日リマインダー」セクションを参照し、本日のごみ収集を判定：
- 水曜・土曜 → 🟠 燃えるごみ
- 第2・第4木曜 → 🔵 燃えないごみ
- リミット：**午前8時まで**

該当日であれば、レポート冒頭に「🚨【本日ごみの日】〇〇ごみ・8時まで」と目立つように記載する。

## ユキちゃん向け業務ブリーフィング生成（2026-04-23 追加）

朝のミーティングレポート作成後、**iPhoneのユキちゃん向けの業務ブリーフィング**をAppleメモに作成・更新する：

### 手順
1. 本日の各プロジェクト進捗・主要タスクを集約
2. 近々の予定（今週〜来週）を抽出
3. マスターの健康状態・重要事項を含める
4. 以下のosascriptでAppleメモに新規ノート作成または更新：

```bash
# 既存の「業務ブリーフィング」ノートを削除
osascript -e 'tell application "Notes" to tell account "iCloud" to delete (notes of folder "Notes" whose name starts with "業務ブリーフィング")'

# 新規ノート作成（本日日付付き）
# タイトル: 業務ブリーフィング_YYYY-MM-DD
# 本文: 下記フォーマット
```

### ブリーフィングのフォーマット
```
# 業務ブリーフィング 2026-04-XX

## 🏛️ 本店からの引き継ぎ

## 📋 現在の主要プロジェクト
- 事業部（あさぎり赤富士商店）: [状況]
- 経理部: [状況]
- 販売部: [状況]
- 保険部門: [状況]

## 📅 近々の予定（3日分）
- YYYY-MM-DD HH:MM タイトル

## 🎯 本日の重点
- 重点事項1
- 重点事項2

## 🌅 ユキちゃんへ
マスターから質問があった際は、この情報を元にお答えしてください。
不明点は「本店のスターシアに確認しますね」と伝えてOKです。
```

### iCloud同期
ノート作成後、iCloudで自動的にiPhoneのメモに同期されます。
マスターがiPhoneのユキちゃん（Claudeアプリ）にそのノートを共有することで、ユキちゃんが業務状況を把握できます。
