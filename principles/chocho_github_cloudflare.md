# 両学長配信ノート: GitHub・Cloudflare編（2026-04-24）

両学長のYouTubeライブ配信より、Claude Codeを使ったアプリ開発・公開時の重要概念を整理。

## 🗃️ GitHubの基本概念

### GitHubとは
- **アプリに特化したファイル置き場**（ネット上の倉庫）
- リポジトリ＝フォルダのようなもの

### リポジトリの2種類

| 種類 | アクセス範囲 | 用途 |
|---|---|---|
| **非公開リポジトリ** | 自分だけが見られる | 開発中・個人利用 |
| **公開リポジトリ** | 全員が見られる | OSS・公開プロジェクト |

### 🛡️ 学長推奨：非公開リポジトリ徹底

初心者は**非公開リポジトリ**で十分。理由：
- ソースコードに API キー・認証情報が含まれる可能性
- 公開すると全世界に流出
- ハッキング被害のリスク
- 個人情報・顧客情報の漏洩

---

## ⚠️ 公開リポジトリの危険性（具体例）

### 漏洩する情報
- **API キー**（OpenAI, Stripe等の課金キー）
- **データベース接続情報**
- **顧客データ**（会員登録系アプリ）
- **営業先リスト・顧客名簿**
- **内部ロジック**（ビジネス上の秘伝のたれ）

### 起こりうる被害
- 不正アクセス・データ改ざん
- 個人情報流出
- 金銭的被害（API キーの不正利用）
- 信用失墜

## 🌐 アプリをみんなに使ってもらうには？

### 🎯 GitHub では公開しない

**GitHubはソースコード保管庫**。アプリ公開には別のサービスが必要。

### 主要なアプリ公開サービス

| サービス | 特徴 | 商用利用 |
|---|---|---|
| **Cloudflare** | リベ推奨、世界最大級 | ✅ OK |
| **Vercel** | 有名・高機能 | ❌ 無料は商用NG |
| **Netlify** | 老舗 | ⭕ 限定的 |

### 🌟 学長推奨：Cloudflare

理由：
- 世界最大級のサーバー会社（規模の経済で強い）
- 無料プランで商用利用OK ✨
- 実レート同等の優れたサービス品質
- リベシティでも統一して使用中

---

## 🔧 Cloudflare の使い方（2層構造）

### 1️⃣ Cloudflare Pages
- 公開用サーバー（ホスティング）
- 静的サイト・SPA等を配信
- カスタムドメイン対応

### 2️⃣ Cloudflare Access
- **アクセス制限**機能
- 特定の人だけアクセス可能に
- 例：「家族にだけ見せたい」「会員限定アプリ」

### 用途別の使い分け

| 用途 | Pages | Access |
|---|---|---|
| 一般公開ホームページ | ✅ | 不要 |
| 会員限定アプリ | ✅ | ✅ 必須 |
| 家族にだけ見せたいアプリ | ✅ | ✅ 必須 |
| 自分だけが使うアプリ | ✅ | ✅ 必須 |

---

## ⚠️ 無料プランの罠（重要）

### Cloudflare の構造
```
GitHubリポジトリ（コード保管）
   ↓ 連携
Cloudflare Pages（公開サイト）
```

### 有料プランの場合
- GitHub と Cloudflare で**別々の権限設定**が可能
- GitHubは非公開、Cloudflareの公開ページは公開、と独立制御OK

### 無料プランの罠
- **GitHub を公開すると Cloudflare も全部公開**
- **GitHub を非公開にすると Cloudflare も非公開**
- 全部まとめてしまう仕組み → 細かい制御不可

### 💡 対策
- **GitHub を非公開リポジトリにする**（鉄則）
- 公開部分はCloudflare Pages 単体で運用
- 公開・非公開を混ぜない

## 🐊 リベシティの呪文（学長配布）

### 概要
リベシティでは、Claude Code（クロコちゃん）に**安全運用ルール**を覚えさせる「呪文」を配布。

### 呪文の主な内容
- GitHub は**非公開リポジトリ徹底**
- API キーは `.env` + `.gitignore` で除外
- 顧客データは絶対に含めない
- 公開時は Cloudflare Access で制限

### 配布場所
- リベシティの学長マガジンに掲載
- Claude Code の設定ファイルに貼り付けて使う

| 必要なもの |
|---|

## 🛡️ 開発者の事故防止チェックリスト

### コードを書く時
- [ ] API キーをコード直書きしていないか
- [ ] `.env` ファイルを `.gitignore` に入れたか
- [ ] 顧客データ・個人情報をコミットしていないか
- [ ] 認証情報をコミット履歴に残していないか

### 公開する前
- [ ] GitHub リポジトリは非公開か
- [ ] Cloudflare Access の制限設定は正しいか
- [ ] テスト用データが残っていないか
- [ ] 本番環境変数を Cloudflare 側で設定したか

### 公開後
- [ ] 動作確認
- [ ] アクセス制限の動作確認
- [ ] エラーログの監視

## 📝 メイド長の運用ルール

マスターが Claude Code でアプリ開発する場合：

1. **必ず非公開リポジトリ**を提案する

3. **公開時は Cloudflare Pages + Access** を提案
4. **顧客データの混入チェック**を行う
5. **学長呪文の概念を踏まえて**運用する
