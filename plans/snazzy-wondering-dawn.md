# ロールバック（復元）システム構築プラン

## Context
同期コマンド（`本店から支店`/`支店から本店`）実行時にファイルが上書きされるが、現状バックアップ機構がない。設定ミスや意図しない上書きから復元できるようにする。

## 仕様
- **保存タイミング**: 同期コマンド実行前（自動）
- **保存条件**: 前回バックアップと内容が違う場合だけ保存
- **保存期間**: 4ヶ月（それより古いものは自動削除）
- **対象ファイル**: `CLAUDE.md`、`settings.json`、スクリプト類（`*.py`、`*.sh`）、`org/`、`scheduled-tasks/`
- **復元方法**: マスターが「ロールバック」と入力 → 一覧表示 → 番号指定で復元
- **保存先**: `~/.claude/backups/snapshots/`

## 作成・変更するファイル

### 1. 新規: `~/.claude/rollback.sh`
バックアップ＆復元スクリプト。3つのモードを持つ：

```
rollback.sh snapshot   # スナップショット取得（同期前に自動実行）
rollback.sh list       # バックアップ一覧表示
rollback.sh restore <日付ディレクトリ名>  # 指定バックアップから復元
```

**snapshot モード:**
1. `~/.claude/backups/snapshots/YYYY-MM-DD_HHMMSS/` にディレクトリ作成
2. 対象ファイルをコピー
3. 前回スナップショットと `diff` で比較 → 差分なければディレクトリ削除（枠を消費しない）
4. 4ヶ月より古いスナップショットを自動削除

**list モード:**
- 保存済みスナップショットの日時と変更ファイル一覧を表示

**restore モード:**
- 指定スナップショットから `~/.claude/` にファイルを復元
- 復元前に現在の状態もスナップショット保存（安全策）

### 2. 変更: `sync_from_local.sh`（iCloud上）
- スクリプト先頭（同期処理の前）に `rollback.sh snapshot` の呼び出しを1行追加

### 3. 変更: `sync_to_local.sh`（iCloud上）
- スクリプト先頭（同期処理の前）に `rollback.sh snapshot` の呼び出しを1行追加

### 4. 変更: `CLAUDE.md`
- 「ロールバック」コマンドの説明を追加（マスターが「ロールバック」と入力したら `rollback.sh list` を実行して一覧表示）

## 検証手順
1. `rollback.sh snapshot` を手動実行 → `~/.claude/backups/snapshots/` にスナップショットが作成されることを確認
2. もう一度 `rollback.sh snapshot` を実行 → 差分なしでスナップショットが削除されることを確認
3. `CLAUDE.md` を少し変更して再度 `snapshot` → 新しいスナップショットが保持されることを確認
4. `rollback.sh list` で一覧が表示されることを確認
5. `rollback.sh restore` で復元できることを確認
6. `本店から支店` 実行時に自動でスナップショットが取られることを確認
