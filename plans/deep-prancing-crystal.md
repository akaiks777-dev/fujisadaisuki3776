# 組織型プロジェクト管理体制の整備

## Context
現在はプロジェクトごとに別セッションで作業しているだけで、PMが各担当を統括する仕組みがない。マスターが直接各セッションに指示を出している状態。これを会社組織のように、PMセッションが各部門を管理する体制に整備する。

## 組織図
```
マスター（指示者）
  └→ PM（メインセッション = このセッション）
        ├→ 事業部（事業計画・輸入販売）
        ├→ 経理部（マネーフォワード家計管理）
        ├→ 販売部（メルカリ出品）
        └→ 新規部門（今後追加）
```

## 実装内容

### 1. 共有ステータスボード作成
**ファイル**: `~/.claude/org/status-board.md`

全セッションが読み書きできる進捗管理ファイル。各部門がタスク完了時にここを更新する。
- 各部門の最新ステータス
- 直近のアクション履歴
- 次のアクション

### 2. 部門別プロンプトファイル作成
**ディレクトリ**: `~/.claude/org/departments/`

各部門の役割・責任範囲・手順を定義したファイル：
- `事業部.md` — 輸入販売の事業計画・調査
- `経理部.md` — マネーフォワード家計管理・分析
- `販売部.md` — メルカリ出品・在庫管理

### 3. PMスキル作成
**ファイル**: `~/.claude/scheduled-tasks/pm-daily-report/SKILL.md`

PMとしての定期レポート機能：
- 毎朝ステータスボードを読み込み
- 各部門の進捗をマスターに報告
- 滞っているタスクのアラート

### 4. CLAUDE.md 更新
組織型管理のルールを更新：
- PMセッションの運用ルール追記
- 各セッション開始時にステータスボード確認のルール
- タスク完了時のステータスボード更新ルール

## 対象ファイル
- `~/.claude/org/status-board.md`（新規作成）
- `~/.claude/org/departments/事業部.md`（新規作成）
- `~/.claude/org/departments/経理部.md`（新規作成）
- `~/.claude/org/departments/販売部.md`（新規作成）
- `~/.claude/scheduled-tasks/pm-daily-report/SKILL.md`（新規作成）
- `~/.claude/CLAUDE.md`（更新）
- `~/Library/Mobile Documents/com~apple~CloudDocs/ClaudeCode_Sync/sync_from_local.sh`（org/ディレクトリを同期対象に追加）

## 検証方法
1. `~/.claude/org/` ディレクトリと各ファイルが正しく作成されたか確認
2. ステータスボードが読み書きできるか確認
3. `CLAUDE.md` の組織管理セクションが正しく更新されたか確認
4. 同期スクリプトに `org/` が追加されたか確認
5. `ミニからノート` で同期が通るか確認
