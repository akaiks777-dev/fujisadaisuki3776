## 「はる」スクリーンショット機能（重要・削除禁止）

ユーザーが `はる` とだけ入力すると、自動でスクリーンショットを撮影してClaudeに見せる機能。

### 仕組み
- `settings.json` の `UserPromptSubmit` フックで `~/.claude/kuro_screenshot.py` を実行
- `kuro_screenshot.py` がプロンプトを監視し、`はる` の場合に `screencapture` でスクリーンショットを `/tmp/kuro_screenshot.png` に保存
- フックが `additionalContext` でClaudeに画像読み込みを指示
- ClaudeはReadツールで `/tmp/kuro_screenshot.png` を読み込んで表示する

### 必須ファイル
- `~/.claude/settings.json` — `UserPromptSubmit` フックに `python3 ~/.claude/kuro_screenshot.py` の記述が必要
- `~/.claude/kuro_screenshot.py` — スクリーンショット撮影スクリプト
- `iCloud: ClaudeCode_Sync/settings.json` — 同じ内容をiCloudにも保存

### settings.json が壊れたら
`iCloud Drive/ClaudeCode_Sync/sync_to_local.sh` を実行して復元する。

## ロールバック（復元）コマンド（重要・削除禁止）

### 概要
`CLAUDE.md`、`settings.json`、スクリプト類、`org/`、`scheduled-tasks/` のバックアップ・復元システム。
同期コマンド実行前に自動でスナップショットを取得。内容に変更がある場合のみ保存。**永久保持**（自動削除なし）。

### 「ロールバック」
ユーザーが `ロールバック` と入力したら、以下を実行：
```
bash ~/.claude/rollback.sh list
```
一覧を表示し、マスターが番号を指定したら復元を実行：
```
bash ~/.claude/rollback.sh restore <番号>
```

### 手動スナップショット
```
bash ~/.claude/rollback.sh snapshot
```

### 必須ファイル
- `~/.claude/rollback.sh` — バックアップ＆復元スクリプト
- `~/.claude/backups/snapshots/` — スナップショット保存先

## メルカリ出品自動化手順（重要・削除禁止）

### 概要
マスターが `iPhone` で商品写真を撮影 → `AirDrop` で `Mac` に転送 → `Claude` が相場調査・出品情報入力を自動で行う

### 手順

#### 1. 写真の準備
- `iPhone` で商品写真を撮影
- `AirDrop` で `Mac` に送信（`~/Downloads/` に保存される）
- `Finder` で写真の場所を開く：`open -R ~/Downloads/写真ファイル名`

#### 2. 相場・販売期間の調査
- メルカリで同一商品の **売り切れ商品** を検索：`https://jp.mercari.com/search?keyword=商品名&status=sold_out`
- 販売価格帯・売れるまでの日数を調査
- **販売中の競合** も検索：`https://jp.mercari.com/search?keyword=商品名&status=on_sale`
- 調査結果をマスターに報告

#### 3. 出品ページの入力（`Chrome` 操作）
- `https://jp.mercari.com/sell/create` を開く
- 写真：マスターが手動でドラッグ&ドロップ（`file_upload` はセキュリティ制限でブロックされる）
- 商品名：`40`文字以内で簡潔に
- カテゴリー：適切なカテゴリーを選択
- 商品の状態：新品なら「新品、未使用」

## 絶対遵守ルール：マスターの環境を壊さない（2026-04-14 説教により制定）

3. **一度成功した手順は正確に再現する** — 余計な回り道・別の方法を試さない
4. **頼まれていないことをしない** — 余計な分析・おすすめは不要。結果だけ出す
5. **特典航空券と言われたら通常航空券やGoogle Flightsで検索しない**
6. **影響がある操作をする前に必ずマスターに報告・確認する** — 事後報告は許されない
7. **記事・投稿で使った画像は絶対に削除しない** — サムネイル・本文画像・生成画像は`~/Downloads/ノウハウ図書館画像/`に保存して永久保持する。Stopフックの自動クリーンアップ対象から除外すること
8. **Chromeのタブは作業完了後に閉じる** — 開いたタブは用が済んだら必ず閉じる。タブを放置しない。マスターの画面を散らかさない

## MacBook Air 支店運用ルール（重要・削除禁止）

### 基本方針
MacBook Air（支店）では**ローカルで全作業を完結**する。Mac mini（本店）への`SSH`接続・GUI遠隔操作は行わない。

### ルール
1. **調査・Chrome操作** → 支店の`MacBook Air`ローカルの`Chrome`で行う（`Claude in Chrome`拡張機能で操作）
2. **本店への接続禁止** → `ssh macmini`による遠隔操作は行わない（銀行口座等のセキュリティ画面が意図せず開くリスクを排除）
3. **データ同期** → 作業完了後に`支店から本店`コマンドで`iCloud`経由で一括同期する
4. **本店の役割** → データの保管場所・同期先・定期タスク（ミーティングレポート等）の実行環境として常時稼働

### セキュリティ上の理由
- 支店から本店を遠隔操作すると、本店側でロック解除が必要になり、銀行口座やログイン画面が意図せず表示されるリスクがある
- 支店内で完結することで、このリスクを完全に排除する

### 判定方法
ホスト名が `macbook` を含む場合、このルールを適用する。

## 議論ボード（本店⇔支店の意見調整）（重要・削除禁止）

### 概要
本店と支店のClaude同士で設定や内容が食い違った場合に、意見を交換して調整する仕組み。

### ファイル
- `~/.claude/discussion_board.md` — 議論ボード本体
- 同期コマンド（`本店から支店` / `支店から本店`）で自動同期される

### 運用ルール
1. **設定がダブっている**・**内容が食い違っている**場合は、議論ボードに理由付きで意見を書く
2. 本店・支店それぞれが「本店」「支店」を明記して書き込む
3. **最終判断は必ずマスターが行う** — Claude同士で勝手に決定しない
4. マスターに「議論ボードに意見の相違があります」と報告し、判断を仰ぐ
5. マスターの判断が出たら、その内容に従って設定・内容を統一する
